1. Introducción

Este manual describe los procedimientos de mantenimiento y soporte técnico para el Sistema de Valoración y Calificación del Desempeño de Docentes y Alumnos (SIVACAD). Está dirigido al personal de soporte técnico y a las desarrolladoras del sistema responsables de atender incidencias, diagnosticar fallas y garantizar la estabilidad operativa de la plataforma.

2. Arquitectura de Logs

2.1 Logs del Backend (Node.js + Morgan)

El servidor Express utiliza Morgan para logging de peticiones HTTP. Los logs incluyen método, ruta, código de estado y tiempo de respuesta. En producción se redirigen a archivo:

node src/server.js > logs/backend.log 2>&1

2.2 Logs de Base de Datos (MySQL)

SET GLOBAL slow_query_log = 'ON';
SET GLOBAL long_query_time = 2;  -- segundos

2.3 Logs de Flask ML

python app.py > logs/flask.log 2>&1

4. Revisión de Sesiones

4.1 Consultar sesiones activas

SELECT u.nombres, u.apellidos, r.nombre_rol, s.fecha_inicio,
       s.ultima_actividad, s.ip_origen, s.estado
FROM sesiones_activas s
JOIN usuarios u ON s.id_usuario = u.id_usuario
JOIN roles r ON u.id_rol = r.id_rol
WHERE s.estado = 'activa'
ORDER BY s.ultima_actividad DESC;

4.2 Cerrar sesión forzadamente

UPDATE sesiones_activas
SET estado = 'expirada', fecha_fin = NOW()
WHERE id_sesion = ?;

4.3 Limpiar sesiones expiradas (CRON)

DELETE FROM sesiones_activas
WHERE estado = 'expirada'
AND fecha_fin < NOW() - INTERVAL 30 DAY;

5. Recuperación de Acceso

5.1 Usuario olvidó su contraseña

Usar ¿Olvidaste tu contraseña? en el login. Si SMTP no funciona, usar modo desarrollo o restablecer en BD:

-- Generar hash bcrypt con: node -e "console.log(require('bcryptjs').hashSync('NuevaPass', 12))"
UPDATE usuarios SET contrasena_hash = 'HASH_AQUI'
WHERE correo_institucional = 'usuario@tesi.edu.mx';

-- También marcar tokens previos como usados
UPDATE password_resets SET used = 1 WHERE id_usuario = (SELECT id_usuario FROM usuarios WHERE correo_institucional = 'usuario@tesi.edu.mx');

5.2 Token JWT expirado

El token expira según JWT_EXPIRES_IN (default: 24h). El usuario debe iniciar sesión nuevamente. Verifique que la fecha/hora del servidor sea correcta.

5.3 Cuenta desactivada

UPDATE usuarios SET estado = 'Activo'
WHERE correo_institucional = 'usuario@tesi.edu.mx';

9. Mantenimiento Preventivo

9.1 Tareas diarias

  • Revisar logs del backend en busca de errores.
  • Verificar que los 3 servidores funcionen.
  • Revisar dashboard de auditoría.
  • Atender incidencias reportadas.

9.2 Tareas semanales

  • Respaldar la base de datos.
  • Limpiar sesiones expiradas.
  • Verificar espacio en disco.
  • Revisar consultas lentas en MySQL.

9.3 Tareas mensuales

  • Actualizar dependencias npm (backend y frontend).
  • Verificar que los modelos ML funcionen correctamente.
  • Revisar integridad de la cadena SHA-256 de auditoría.
  • Realizar pruebas de carga ligeras.

9.4 Comandos de mantenimiento

# Respaldar BD
mysqldump -u root -p sivacad_isc > backups/sivacad_isc_$(date +%Y%m%d).sql

# Limpiar temporales
find backend/uploads/actas-ocr/ -type f -mtime +30 -delete

# Actualizar dependencias
cd backend && npm update
cd frontend && npm update

# Verificar vulnerabilidades
cd backend && npm audit
cd frontend && npm audit

10. Procedimientos de Recuperación

10.1 Recuperación de BD

mysql -u root -p sivacad_isc < backups/sivacad_isc_20260722.sql

10.2 Recuperación de uploads

Los archivos subidos están en backend/uploads/. Si se pierden, los usuarios deberán subirlos nuevamente. Los datos de la BD (rutas) se conservan.

10.3 Recuperación tras caída del servidor

  1. Reiniciar servicios en orden: MySQL → Backend → Frontend → Flask ML.
  2. Ejecutar script de diagnóstico.
  3. Verificar que todos los servicios respondan.
  4. Notificar a los usuarios.

10.4 Recuperación de acceso de administrador

-- Si no hay admin accesible:
UPDATE usuarios SET estado = 'Activo' WHERE id_rol = 1 LIMIT 1;

-- Si no hay admins, crear uno (requiere hash bcrypt):
-- node -e "console.log(require('bcryptjs').hashSync('Admin123!', 12))"
INSERT INTO usuarios (nombres, apellidos, correo_institucional,
  contrasena_hash, estado, id_rol)
VALUES ('Admin', 'Recuperado', 'admin@tesi.edu.mx',
  '$2a$12$...hash...', 'Activo', 1);

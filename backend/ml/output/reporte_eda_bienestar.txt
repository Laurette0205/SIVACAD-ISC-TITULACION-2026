   • 1 check-in, 1 alerta(s), 1 sesión(es), 6 mensaje(s)
   • 0 derivaciones — consistentes con nivel de riesgo Crítico

2. DIMENSIONES CAPTURADAS (9 variables escala 1-10):
   • Ánimo, Energía, Sueño, Estrés, Apoyo, Ambiente,
   • Carga Académica, Carga Laboral, Enfoque
   • Además: bienestar_score global, índice_riesgo compuesto

3. POTENCIAL PARA ML EN BIENESTAR:
   • Clasificación: predecir nivel_riesgo (Bajo/Medio/Alto/Crítico) desde las 9 dimensiones
   • Regresión: predecir bienestar_score continuo
   • Detección de anomalías: identificar patrones atípicos previos a crisis
   • Alertas tempranas: clasificar si requiere_derivacion = True

4. LIMITACIONES:
   • 1 solo registro → insuficiente para entrenar cualquier modelo
   • No hay historial temporal para detectar tendencias de bienestar
   • No hay datos de crisis (derivaciones=0) para aprendizaje de casos graves

5. RECOMENDACIÓN:
   • Implementar check-ins semanales obligatorios para acumular datos
   • Una vez con 200+ registros, entrenar clasificador de nivel_riesgo
   • Integrar con el sistema de alertas para derivación automática ML-based
